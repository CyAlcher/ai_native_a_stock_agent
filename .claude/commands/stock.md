## 用法

```
/stock 我现在持有五粮液000858，跌了3个点，不知道要不要补仓
/stock 股票名称=贵州茅台 股票代码=600519
/stock                    ← 不带参数，显示节点选择菜单
```

## A股交易全周期节点地图

```
空仓观望 → 信息搜集 → 分析决策 → 买入建仓
                                      ↓
等待下一次机会 ← 复盘总结 ← 离场执行 ← 触发止损止盈
                                      ↑
                          持仓波动 → 盈利管理
                                  → 亏损管理
```

| 节点 | 命令 | 适用场景 |
|------|------|---------|
| 空仓观望 | `/stock/watch` | 空仓，不知道要不要进场 |
| 信息搜集 | `/stock/research` | 想了解一只票的基本情况 |
| 分析决策 | `/stock/analyze` | 纠结要不要买、买在哪 |
| 买入建仓 | `/stock/buy` | 准备建仓，确定买入方式和止损位 |
| 持仓波动 | `/stock/hold` | 已持仓，股价出现波动 |
| 盈利管理 | `/stock/profit` | 持仓盈利，纠结止盈或加仓 |
| 亏损管理 | `/stock/loss` | 持仓亏损，纠结割肉或补仓 |
| 触发止损止盈 | `/stock/trigger` | 股价到了止损/止盈位 |
| 离场执行 | `/stock/exit` | 准备卖出，处理离场后事宜 |
| 复盘总结 | `/stock/review` | 交易结束，总结经验教训 |
| 等待下一次机会 | `/stock/wait` | 离场后等待下次入场机会 |

## Outline

1. **解析 $ARGUMENTS**

   从用户输入中提取：
   - `股票名称`（如"五粮液"、"茅台"）
   - `股票代码`（如"000858"、"600519"）
   - 交易状态描述（如"跌了3个点"、"空仓"、"刚买入"）

2. **判断当前节点**

   根据交易状态描述，匹配最合适的节点：

   | 关键词 | 推荐节点 |
   |--------|---------|
   | 空仓、没买、要不要进 | 空仓观望 |
   | 了解、研报、公告、为什么涨 | 信息搜集 |
   | 要不要买、能买吗、支撑压力 | 分析决策 |
   | 建仓、怎么买、止损位 | 买入建仓 |
   | 持仓、不动了、洗盘、补仓 | 持仓波动 |
   | 赚了、盈利、要不要卖、加仓 | 盈利管理 |
   | 亏了、跌了、割肉、装死 | 亏损管理 |
   | 破线了、到目标价了、减持公告 | 触发止损止盈 |
   | 卖完了、清仓了、资金去哪 | 离场执行 |
   | 复盘、总结、年化收益 | 复盘总结 |
   | 手痒、等机会、换策略 | 等待下一次机会 |

3. **无法判断时**

   列出所有节点让用户选择，格式：
   ```
   你现在处于哪个交易阶段？
   1. 空仓观望   2. 信息搜集   3. 分析决策   4. 买入建仓
   5. 持仓波动   6. 盈利管理   7. 亏损管理   8. 触发止损止盈
   9. 离场执行  10. 复盘总结  11. 等待下一次机会
   ```

4. **跳转到对应节点 command**

   确定节点后，调用对应的 `/stock/<slug>` command，并传入已提取的股票参数。
